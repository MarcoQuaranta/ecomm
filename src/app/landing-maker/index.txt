================================================================================
                         LANDING MAKER - ISTRUZIONI PRINCIPALI
================================================================================

Quando l'utente dice "vai in landing-maker", devi eseguire questo processo.

IMPORTANTE: Ogni fase richiede l'approvazione dell'utente prima di procedere
alla successiva. NON passare alla fase successiva senza OK esplicito.

REGOLA FONDAMENTALE: Ogni landing e' composta da 3 file. NON creare MAI
solo il content.json. Crea SEMPRE tutti e 3 i file insieme:
  1. page.tsx       (importa il template del venditore + content.json)
  2. content.json   (copy, prezzi, immagini, tracking)
  3. ty/page.tsx    (importa il template TY del venditore + content.json)

================================================================================
FASE 0 - RACCOLTA DATI PRODOTTO
================================================================================

Chiedi all'utente:

1. "Per quale venditore devo creare la landing?" (es: Mirko, Lino, Daniele)
   NOTA: ogni venditore ha il suo template dedicato. Se il template non esiste
   ancora, avvisa l'utente e chiedi come procedere.
   Template disponibili:
   - Mirko: LandingTemplateMirko + TyTemplateMirko (landing-mirko.css)
   - (altri template verranno aggiunti qui man mano)

2. "Qual e' il nome del prodotto?"

3. "Qual e' l'angle principale? (es: risparmio, comodita', salute, innovazione...)"

4. "Inserisci i dati aggiuntivi sul prodotto (link, info, specifiche, tutto quello
   che hai):"

5. "Hai note aggiuntive, preferenze o cose da tenere a mente per questa landing?"

Dopo aver raccolto TUTTE le info (incluse le note aggiuntive), procedi alla Fase 1.

--> ATTENDI OK UTENTE

================================================================================
FASE 1 - ANALISI DATI + CREAZIONE LANDING ITALIANA
================================================================================

Leggi il file: fase1-analisi-copy.txt

Il tuo compito:
1. Leggi template-json.json per la struttura (regole STATIC/DYNAMIC)
2. Analizza i dati ricevuti dall'utente
3. Crea il content.json rispettando le sezioni STATIC/DYNAMIC
4. Crea page.tsx (importa il template del venditore)
5. Crea ty/page.tsx (importa il template TY del venditore)
6. Crea le cartelle immagini (vuote)

Output: 3 file creati + cartella immagini. Landing subito visibile nel browser.

--> MOSTRA IL COPY CREATO (sezioni DYNAMIC)
--> COMUNICA L'URL PER VEDERE LA LANDING
--> CHIEDI: "Copy completato. Vuoi modifiche o approvi?"
--> ATTENDI OK UTENTE (o applica modifiche richieste)

================================================================================
FASE 2 - CARTELLE PER ALTRI MEDIA BUYER (opzionale)
================================================================================

Leggi il file: fase2-creazione-cartelle.txt

Se ci sono altri media buyer oltre a quello della Fase 1, crea le landing
adattando il copy allo stile di ciascun venditore. Se il venditore e' uno
solo, SALTA questa fase.

Per ogni media buyer aggiuntivo, crea TUTTI E 3 i file + cartella immagini.

ESECUZIONE PARALLELA: Lancia un agente per ogni media buyer.

--> MOSTRA RIEPILOGO CARTELLE CREATE
--> CHIEDI: "Cartelle create. Controllale e dimmi cosa modificare."
--> ATTENDI OK UTENTE (prima approvazione)

Dopo approvazione dell'utente:
1. PUSH SU GITHUB (chiedi su quale account pushare, segui le regole in CLAUDE.md)
2. CHIEDI: "Push fatto. Ora attendo l'approvazione del media buyer. Dimmi quando ha approvato."
3. ATTENDI SECONDA APPROVAZIONE (dal media buyer, tramite l'utente)
4. Solo dopo la seconda conferma, procedi alla Fase 3.

================================================================================
FASE 3 - TRADUZIONI
================================================================================

Leggi il file: fase3-traduzioni.txt

Questa fase si attiva SOLO dopo approvazione della Fase 2 (o Fase 1 se c'e'
un solo venditore).

1. Chiedi: "In quali nazioni vuoi tradurre?"
2. Chiedi: "Qual e' il prezzo di vendita per ogni nazione?" (valuta locale)
3. Per ogni nazione, crea TUTTI E 3 i file:
   - page.tsx: COPIATO identico dalla versione italiana
   - content.json: TRADOTTO nella lingua target
   - ty/page.tsx: COPIATO identico dalla versione italiana
   - Adatta nomi/citta' testimonianze al paese
   - Inserisci prezzo e valuta locale

Struttura risultante:
src/app/[venditore]/[prodotto]/
  ├── [prodotto]-ita/      <-- originale
  ├── [prodotto]-pl/       <-- traduzione polacca
  ├── [prodotto]-hu/       <-- traduzione ungherese
  └── ...

--> MOSTRA RIEPILOGO TRADUZIONI
--> CHIEDI APPROVAZIONE

================================================================================
FASE 4 - TRACCIAMENTO
================================================================================

Leggi il file: fase4-tracciamento.txt

Questa fase si attiva SOLO dopo approvazione della Fase 3.

1. Chiedi: "Dammi i dati di tracciamento Google Ads (ID + conversion label)"
2. Chiedi: "Dammi il pixel ID di Meta Ads"
3. Chiedi: "Dammi i dati del network (API endpoint, offer ID, ecc.)"
4. Inserisci il tracciamento in TUTTI i content.json di ogni nazione
5. Il valore di conversione e' dinamico = prezzo prodotto in valuta locale
6. Verifica che tutto sia corretto
7. Report finale con check completo

--> MOSTRA REPORT FINALE
--> CHIEDI CONFERMA

================================================================================
STORICO
================================================================================

Aggiorna sempre il file storico.json con la nuova landing creata.

================================================================================
RIEPILOGO FLUSSO
================================================================================

FASE 0 (raccolta dati prodotto)           -->  OK utente  -->
FASE 1 (analisi + landing italiana)       -->  OK utente  -->
FASE 2 (cartelle altri media buyer)       -->  OK utente  -->  PUSH GITHUB  -->  OK media buyer  -->
FASE 3 (traduzioni per nazione + prezzi)  -->  OK utente  -->
FASE 4 (tracciamento Google/Meta/network) -->  validazione  -->  report finale
