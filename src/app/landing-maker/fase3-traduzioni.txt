================================================================================
                    FASE 3 - TRADUZIONI
================================================================================

PREREQUISITI:
- Tutte le landing italiane devono essere approvate dall'utente
- L'utente deve aver dato l'OK finale sulla Fase 2

================================================================================
STEP 1 - RACCOLTA INFO
================================================================================

Chiedi all'utente:

1. "In quali nazioni vuoi tradurre?" (le nazioni NON sono fisse, cambiano ogni volta)

2. "Qual e' il prezzo di vendita per ogni nazione?"
   Specifica che serve: prezzo originale + prezzo scontato, in valuta locale.

3. "Per ogni nazione, su quali piattaforme? (Google Ads / Facebook / entrambe)"

ESEMPIO risposta utente:
- Polonia: 299 PLN -> 149 PLN (GG + FB)
- Ungheria: 29990 HUF -> 14990 HUF (solo GG)
- Rep. Ceca: 2990 CZK -> 1490 CZK (solo GG)

================================================================================
STEP 2 - STRUTTURA CARTELLE
================================================================================

Per ogni nazione, per ogni media buyer, per ogni piattaforma, crea una cartella
con suffisso -[lang]-[piattaforma]:

src/app/[mediabuyer]/[prodotto]/[prodotto]-[lang]-[gg|fb]/
  ├── page.tsx                      <-- COPIATO identico dalla versione italiana
  ├── content.json                  <-- TRADOTTO (TUTTI i campi visibili)
  ├── LandingTemplate[Venditore].tsx  <-- COPIATO identico dalla versione italiana
  ├── TyTemplate[Venditore].tsx       <-- COPIATO identico dalla versione italiana
  ├── landing-[venditore].css         <-- COPIATO identico dalla versione italiana
  └── ty/
      └── page.tsx                  <-- COPIATO identico dalla versione italiana

ESEMPIO per prodotto "cloudrelief", media buyer "mirko", PL (GG+FB) e CS (solo GG):

src/app/mirko/cloudrelief/
  ├── cloudrelief-ita/          <-- originale (test)
  ├── cloudrelief-pl-gg/        <-- polacco Google Ads
  │   ├── page.tsx
  │   ├── content.json          <-- tradotto, tracking GG
  │   └── ty/page.tsx
  ├── cloudrelief-pl-fb/        <-- polacco Facebook
  │   ├── page.tsx
  │   ├── content.json          <-- tradotto, tracking FB
  │   └── ty/page.tsx
  └── cloudrelief-cs-gg/        <-- ceco solo Google Ads
      ├── page.tsx
      ├── content.json          <-- tradotto, tracking GG
      └── ty/page.tsx

### COME CREARE LE VARIANTI PER PIATTAFORMA
Se una nazione va su ENTRAMBE le piattaforme:
1. Crea la versione -gg con il content.json tradotto
2. Copia TUTTI i file dalla -gg alla -fb
3. Nel content.json della -fb: svuota googleAdsId e conversionLabel,
   inserisci facebookPixelId, svuota network (credenziali FB diverse)
4. Il tracking finale viene compilato in Fase 4

================================================================================
STEP 3 - COSA TRADURRE
================================================================================

### REGOLA FONDAMENTALE — ZERO TESTO HARDCODATO:
TUTTI i testi visibili a schermo DEVONO provenire dal content.json.
Il componente (LandingTemplate*.tsx / TyTemplate*.tsx) NON DEVE contenere
MAI testi in nessuna lingua — legge TUTTO dal JSON.
Se trovi un testo hardcodato nel componente, SPOSTALO nel JSON prima di tradurre.

Si traducono SOLO i testi visibili a schermo e gli attributi leggibili dai
motori di ricerca (alt delle immagini, SEO title/description, aria-label).
NON si traduce codice, struttura, link o configurazioni.

### TRADURRE:

1. CONTENT.JSON - SOLO i testi visibili a schermo:
   - Headline, subheadline
   - Benefici (titolo + descrizione)
   - Features (titoli + descrizioni)
   - FAQ (domande + risposte)
   - Testi urgenza/scarsita'
   - Garanzia (titolo + descrizione)
   - Form (titolo, sottotitolo, label, placeholder, bottone, note)
   - CTA (testi pulsanti, sottotesti)
   - Badge e label vari
   - Comparison: featureColumn, winnerBadge, productColumn, competitorColumn, rows
   - Reviews: verifiedBadge, sellerReplyLabel, showMoreText, leaveReviewText
   - Sizes (se presente): sizeLabel, title, options[].name, explanationTitle, explanationText
     NOTA: gli ID delle opzioni sizes (options[].id e defaultSelected) vanno adattati
     alla lingua target per coerenza (es: "singolo" → "jednoosobowy" in polacco)
   - ariaLabels: prevSlide, nextSlide, thumbnail, close
   - SEO (title, description)
   - Sezione TY (TUTTI i campi visibili: title, subtitle, orderNumberLabel,
     summaryTitle, accessoriesLabel, totalLabel, paymentNote, yourDataTitle,
     nameLabel, phoneLabel, addressLabel, nextStepsTitle, message, steps[], footerText)
   - Alt delle immagini (attributo "alt" nei slides e features)

2. TESTIMONIANZE - tradurre + adattare:
   - Traduci il testo della recensione nella lingua target
   - CAMBIA il nome con un nome TIPICO del paese target (vedi lista sotto)
   - CAMBIA le iniziali (initials) per corrispondere al nuovo nome
   - CAMBIA la citta' con una citta' REALE del paese target
   - Traduci anche le risposte del venditore (sellerReply)
   - NON tradurre il nome del prodotto dentro le recensioni

3. PREZZI E VALUTA:
   - Usa i prezzi forniti dall'utente per quella nazione
   - Aggiorna pricing.currency, pricing.currencySymbol, pricing.originalPrice, pricing.currentPrice
   - Ricalcola pricing.discountPercent se necessario
   - Aggiorna TUTTI i testi che contengono prezzi (priceBox.discountLabel, comparison, package, orderForm, stickyCta, midCta)
   - Il valore di conversione (tracking.conversionValue) = prezzo scontato locale
   - tracking.conversionCurrency = valuta locale

4. TY/PAGE.TSX:
   - Il componente TyTemplate legge TUTTI i testi dalla sezione "ty" del content.json
   - NON ci sono testi hardcodati nel componente — tutti vengono dal JSON
   - Assicurati che TUTTI i campi nella sezione "ty" siano tradotti:
     title, subtitle, orderNumberLabel, summaryTitle, accessoriesLabel,
     totalLabel, paymentNote, yourDataTitle, nameLabel, phoneLabel,
     addressLabel, nextStepsTitle, message, steps (array di 3 stringhe), footerText
   - NON toccare il codice, import, componenti, logica del ty/page.tsx

### NON TRADURRE (MAI):

1. PAGE.TSX - copialo IDENTICO dalla versione italiana, NON TOCCARLO
2. NOME PRODOTTO - resta UGUALE in tutte le lingue (es: "QuickChef Pro™" resta cosi')
3. MARCHI E TRADEMARK - simboli ™, ®, nomi brand restano invariati
4. CODICE - variabili, funzioni, import, classi CSS, event handlers, componenti React
5. LINK IMMAGINI - tutti i path /images/ restano IDENTICI (le immagini sono condivise)
6. URL E PATH - link API, endpoint network, path pagine
7. DATI TECNICI - ID tracking, chiavi API, pixel ID, conversion label, configurazioni
8. STRUTTURA JSON - le chiavi del JSON restano in inglese (mai tradurre "title", "description", "items" ecc)
9. CAMPI _note E _instructions - sono commenti per l'AI, restano in italiano

================================================================================
STEP 6 - ESECUZIONE
================================================================================

Lancia agenti in parallelo (uno per media buyer).
Ogni agente traduce in TUTTE le nazioni richieste.

Per ogni traduzione:
1. Crea cartella [prodotto]-[lang]/
2. Copia TUTTI i file dalla versione italiana:
   - page.tsx (identico)
   - LandingTemplate[Venditore].tsx (identico)
   - TyTemplate[Venditore].tsx (identico)
   - landing-[venditore].css (identico)
   - ty/page.tsx (identico)
3. Traduci content.json — OGNI testo visibile, NESSUNA eccezione:
   - Tutti i testi delle sezioni principali
   - Sezione ty (TUTTI i 15+ campi)
   - Sezione comparison (featureColumn, winnerBadge)
   - Sezione reviews (verifiedBadge, sellerReplyLabel, showMoreText, leaveReviewText)
   - Sezione sizes (sizeLabel, title, options names, explanations, IDs adattati)
   - Sezione ariaLabels (prevSlide, nextSlide, thumbnail, close)
4. Inserisci prezzi e valuta locale
5. Adatta nomi/citta' testimonianze
