================================================================================
REGOLE FONDAMENTALI DEL CONTENT.JSON
================================================================================

### SEZIONI STATIC (NON TOCCARE MAI)
Queste sezioni vanno COPIATE IDENTICHE dal template-json.json:
- topBar
- urgencyBanner
- cta (tutti i campi)
- trustBadges
- faq (domande e risposte)
- reviewPopup
- ty (title, subtitle, message)
- Titoli fissi: problems.eyebrow, problems.title, features.eyebrow,
  comparison.eyebrow, comparison.title, package.eyebrow

### SEZIONI SEMI-STATIC (modificare SOLO i valori indicati)
- midCta: cambiare SOLO la percentuale di sconto
- stickyCta: cambiare SOLO i prezzi (derivati da pricing)
- orderForm: cambiare SOLO productName e prezzi (derivati da pricing)

### SEZIONI DYNAMIC (da generare per il prodotto)
- seo (title, description)
- pricing (originalPrice, currentPrice, discountPercent, currency, currencySymbol)
- hero (title, highlightedText, subtitle, socialProof)
- gallery (slides con path immagini)
- benefits (6 benefici - SOLO sul prodotto, MAI spedizione/pagamento/garanzia)
- priceBox (productName, tagline, discountLabel, originalPrice, currentPrice, features)
- problems.items (3 problemi)
- features.items (3-4 feature con immagini/GIF angle)
- howItWorks (3 step)
- specs.items (specifiche tecniche)
- comparison.rows (righe confronto)
- package.items (cosa include il pacco)
- reviews (10 recensioni: 9x5 stelle + 1x4 stelle, seller reply su 9/10)

### REGOLA EMOJI / ICONE (FONDAMENTALE)
- MAI usare emoji o emoticon nel content.json. ZERO emoji, in NESSUN campo.
- Per i campi "icon" (trustBadges, problems, package), usare NOMI di icone SVG
  (es: "cash", "truck", "shield", "box", "leaf", "back-pain", "insomnia").
  Il componente renderizza automaticamente icone SVG professionali.
- Per i campi di testo (timerLabel, stockWarning, paymentBadge, reassurance, ecc),
  NON inserire nessuna emoji. Il componente aggiunge le icone SVG dove servono.
- Icone disponibili: cash, truck, return, bone, foot, fatigue, box,
  leaf, droplet, shield, gift, lock, warning, timer, money, check, phone, pencil,
  stiffness, pain, cable.
- Se serve un'icona non in lista, usare quella piu' simile o chiedere all'utente.

### REGOLE PREZZI
- TUTTI i prezzi nel JSON derivano da pricing.originalPrice e pricing.currentPrice
- MAI hardcodare prezzi diversi in altre sezioni
- Il formato e': "69 Euro" (numero + simbolo dalla sezione pricing)

### REGOLE RECENSIONI
- 10 totali: 9 da 5 stelle, 1 da 4 stelle
- Seller reply su 9 recensioni su 10
- Tono informale, colloquiale, realistico
- NO tag HTML nel testo
- NO riferimenti a nazioni specifiche
- Nomi e citta' italiane (per la versione ITA)

### REGOLE SOCIAL PROOF
- hero.socialProof.count DEVE corrispondere a reviews.totalCount
- hero.socialProof.rating DEVE corrispondere a reviews.rating

================================================================================
STEP 1 - ANALISI (INTERNA)
================================================================================

Analizza i dati ricevuti e identifica:

1. PUNTI DI FORZA - Benefici principali, cosa lo rende diverso
2. TARGET - A chi si rivolge, pain points, desideri
3. OBIEZIONI - Perche' non comprerebbero, come anticiparle
4. ANGLE - Usa quello fornito dall'utente + 2-3 secondari

NON creare file separati. L'analisi e' solo interna al tuo processo.

================================================================================
STEP 2 - CREAZIONE CONTENT.JSON
================================================================================

1. Parti dal template-json.json come base
2. COPIA tutte le sezioni STATIC senza modifiche
3. GENERA le sezioni DYNAMIC con il copy del prodotto
4. AGGIORNA le sezioni SEMI-STATIC (solo i valori permessi)
5. Rimuovi tutti i campi _note e _instructions dal JSON finale
6. Verifica che il JSON sia valido

STILE COPY (per versione italiana):
- Marketing aggressivo, semplice, emozionale
- Target boomer-friendly
- Grammatica italiana corretta (accenti, non apostrofi)
- Mai menzionare nazioni specifiche

================================================================================
STEP 3 - CREAZIONE FILE LANDING
================================================================================

IMPORTANTE: Ogni landing DEVE avere SEMPRE 3 file. Non creare MAI solo il JSON.

Per il venditore specificato in Fase 0, crea:

src/app/[venditore]/[prodotto]/[prodotto]-ita/
  ├── page.tsx          <-- importa il template del venditore + content.json
  ├── content.json      <-- il copy generato
  └── ty/
      └── page.tsx      <-- importa il template TY del venditore + content.json

### PAGE.TSX (per Mirko):
```tsx
import LandingTemplateMirko from '@/components/LandingTemplateMirko';
import content from './content.json';

export default function Page() {
  return <LandingTemplateMirko content={content} />;
}
```

### TY/PAGE.TSX (per Mirko):
```tsx
import TyTemplateMirko from '@/components/TyTemplateMirko';
import content from '../content.json';

export default function TyPage() {
  return <TyTemplateMirko content={content} />;
}
```

Per altri venditori, sostituisci il nome del componente:
- Lino: LandingTemplateLino / TyTemplateLino
- Daniele: LandingTemplateDaniele / TyTemplateDaniele

### CARTELLA IMMAGINI:
Crea anche le cartelle per le immagini (vuote, l'utente le riempira'):

public/images/[venditore]/[prodotto]/
  ├── carosello/    <-- foto prodotto: 1.webp, 2.webp, ecc.
  └── angle/        <-- foto feature: 1.webp, 2.webp, 3.webp, 4.webp
